# `jj` 兼容入口

`jj` 用来接住 `$jj`、`/jj` 或“用 jj-flow 处理”这类没有明确子命令的请求，再把原始需求路由到合适的 `jj-*` 工作流。它不是终端里的交付命令，也不是一个会自行替代 Maestro 的执行引擎。

## 适用场景

- 你知道要用 jj-flow，但还没有决定是交付、审查还是迁移。
- 旧提示词或团队约定仍使用 `$jj` / `/jj`，需要兼容现有用法。
- 你希望先保留原始动机和资料，再由 Agent 选择最窄的工作流入口。

## 何时不用

- 已经确定是端到端交付时，直接用 [`$jj-delivery`](command-jj-delivery.html) 或 `/jj-delivery`。
- 已经确定是交付前审查时，直接使用 [`$jj-delivery`](command-jj-delivery.html)，并说明只审查不扩改。
- 需要安装命令资产或为自动化生成结构化调度结果时，使用 [`jj` CLI](command-cli.html)，不要在 shell 中运行 `jj-delivery` 等名字。
- 同源项目迁移或独立控制项目调度应直接使用 `$jj-same`、`$jj-dispatch`，不要把它们伪装成普通交付。

## 输入模板

Codex：

```text
$jj
目标：<要解决的问题或要交付的结果>
资料：<PRD、接口、设计、日志、diff、线程或文件路径>
范围：<本次做什么，不做什么>
验收：<什么结果算完成>
关键决策：<只有你能拍板的取舍，可省略>
```

Claude Code：

```text
/jj <同样的自然语言输入>
```

不需要把资料拆成 `--prd`、`--api`、`--design` 等固定参数。

## 完整示例

### 示例 1：需求跨分析、开发和验收

```text
$jj
目标：完成 AI 获客列表和详情页，并按设计稿做最后精修。
资料：PRD 在 docs/v17.1，接口看 YApi 项目 100591，设计稿是 MasterGo 链接。
范围：本期不做导出和批量操作。
验收：字段来自真实接口，页面覆盖空态和错误态，目标测试与质量审查通过。
```

预期路由到 `$jj-delivery`，因为目标需要资料发现、计划、实现、验证和审查的完整链路。

### 示例 2：线上异常需要定位根因

```text
/jj
现象：goods-detail 页面今天 09:30 到 10:00 间歇性返回 500。
资料：ARMS 指纹为 TypeError: Cannot read properties of undefined，影响 URL 为 /goods-detail。
范围：只修复已证明的根因，不重构商品详情模块。
验收：复现路径消失，相关旧流程回归通过，并说明引入位置。
```

预期路由到 `/jj-delivery`，先查真实日志和代码证据，再决定最小修复。

## 执行过程

1. 保留你的原始需求、动机、范围和资料，不先改写成固定 CLI 参数。
2. 判断任务属于 `delivery`、`validate`、`evolve`、`same`、`dispatch`；功能、线上修复、交付前审查与不确定意图归入 `delivery`。`dispatch` 只在 Codex 环境可用。
3. 读取项目资料、`.workflow` 状态、接口、设计、日志和历史线程等可用证据；**禁止调用 `maestro explore`**。
4. 转入对应的原生 `jj-*` 工作流；需要通用能力时再调用 Maestro/Codex skills。
5. 只有会改变交付结果的缺失信息才询问你；其余缺口标为 `PENDING`。

## 输出/完成标准

- 明确说明最终选择了哪个入口以及选择理由。
- 原始需求、范围和不做范围没有在路由过程中丢失。
- 找到的证据与缺失证据可区分，没有证据的结论不写成 `PASS`。
- 已进入对应工作流并按该工作流的完成标准继续，而不是只给一段命令推荐后停止。
- 如果当前条件不足，输出具体的 `PENDING` 项和解除条件。

## 常见误区

- 把 `$jj` 当作长期主入口。任务类型明确时，直接使用 `$jj-delivery` 等命令更清楚。
- 在终端执行 `jj delivery ...`，期待它直接修改业务代码。CLI 只用于安装、维护和调试输出。
- 为了适配模板而删掉业务动机、范围或“不做什么”。路由必须保留原始语义。
- 资料不足时要求强行给结论。jj-flow 的规则是证据不足保持 `PENDING`。

## 相关命令

- [`jj-delivery`](command-jj-delivery.html)：少参数端到端交付的默认入口（含功能、修复、审查与不确定意图）。
- [`jj` CLI](command-cli.html)：维护与自动化调试入口。
