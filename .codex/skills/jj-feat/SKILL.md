# jj-feat

## 适用场景

功能目标明确，资料相对完整，主要工作是按现有项目模式实现和验证。

## 执行步骤

1. 明确功能目标、影响模块、接口资料和验收标准。
2. 查找相邻实现和项目约定，优先复用既有模式。
3. 给出最小实现计划后改代码。
4. 运行聚焦测试和必要的静态检查。
5. 输出修改范围、验证结果和剩余风险。
6. 功能修改验证通过后，将当前仓库交给 `$jj-same` 核对项目与分支、列出可同步项目并询问同步决策；没有候选时直接结束。

## 边界

不要把功能需求升级成完整重构；没有证据的接口和设计结论保持 `PENDING`。
