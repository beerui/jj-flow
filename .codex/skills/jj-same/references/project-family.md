## 逻辑矩阵

项目族按产品形态和品牌/业务线形成矩阵：

| 形态 | 承接 | 兑接 | 承载 |
|---|---|---|---|
| 前台 | 承接前台 | 兑接前台 | 承载前台 |
| 后管 | 承接后管 | 兑接后管 | 承载后管 |

任意一个单元格都可能成为源项目。不要固定认为承接永远是源；以用户指定的会话、需求落地仓库或功能分支为源。

## 角色判断

业务角色由以下证据确定，优先级从高到低：

1. 用户明确指定“承接/兑接/承载 + 前台/后管”。
2. 项目级架构文档、部署配置和业务入口。
3. 仓库路径、包名和 README，只作为辅助线索。

不要使用以下单一线索重新分类项目：

- Vue 2 或 Vue 3。
- Vuex 或 Pinia。
- Vue CLI 或 Vite。
- 接口路径是否包含 `/api/admin`。
- 仓库名中是否包含 `manager`、`admin`、`frontend`。

技术形态决定 `DIRECT` 还是 `ADAPT`，不决定业务角色。

## 迁移方向

### 同一行迁移

- 前台需求默认只在三个前台项目中分析。
- 后管需求默认只在三个后管项目中分析。
- 同源不等于同构；每个目标仍需建立调用链和能力矩阵。

### 跨行迁移

只有满足以下任一条件才分析前台与后管之间的同步：

- 用户明确要求前台和后管同时修改。
- 需求文档明确列出两种形态。
- 共享接口契约或安全规则明确要求全端一致。

即使跨行共享业务规则，也要分别用前台和后管的原生架构实现，不能复制组件。

## 默认交付顺序

当承接项目是用户指定的领头项目时，同一行默认按 `cj -> dj -> cz` 串行交付：

1. `cj` 完成分析、开发、验证与评审。
2. 用户在新会话中引用前一会话 ID，明确触发 `dj`。
3. `dj` 重新做目标分析，按自身业务和架构实施并完成验证与评审。
4. 用户再次主动触发后，才进入 `cz`。

前置项目没有稳定 commit、验证未通过、评审阻塞、存在影响 `MUST` 的未解决项，或家族交付计划尚未更新时，下一个项目保持 `BLOCKED`。不得并行提前修改后续项目，也不得由 agent 自行进入另一个仓库继续开发。

`cj -> dj -> cz` 是承接领头时的默认顺序，不是永久角色限制。当前项目不是承接、用户指定其它顺序或需求只覆盖部分项目时，以用户当前要求和任务级登记为准；不要倒推补做未授权项目。

## 分支派生规则

- 领头项目分支由用户创建，`jj-same` 只验证并记录，不替用户创建或改名。
- 后续项目仅在前置门禁通过且用户主动触发后，从该项目本地 `master` 创建开发分支。不要自动 pull、更新或改写本地 `master`。
- 后续分支沿用领头分支的类型、发布日期和任务序号，只替换角色前缀：`feat/cj-0717-1 -> feat/dj-0717-1 -> feat/cz-0717-1`。
- 任务序号表示同一发布日期下的任务编号，跨项目同步同一任务时保持一致。`-1 / -2` 不能用来表示 `dj / cz` 的项目顺序。
- 领头分支不符合可判定格式时，先从家族计划、会话和 Git 历史验证。能够按“不扩大范围、只替换角色前缀”得到唯一可回退结果时记录假设并继续；仍存在多个有效解释时标记 `BLOCKED`，记录候选分支、缺失证据和解除条件，不启动额外问答流程。
- 创建前重新检查目标 repo、origin、工作区、本地 `master` SHA 和目标分支是否已存在；任一事实冲突时标记 `BLOCKED`。

### 六项目根因检查

用户要求“检查所有项目是否有同类问题”时：

1. 先在源项目定位根因和触发条件。
2. 在六个项目逐一寻找同一根因，不以相似症状代替。
3. 只修存在同一根因且属于需求范围的项目。
4. 将其余项目标记为 `N/A`、`BLOCKED` 或“未发现同根因”。

## 任务级项目登记

每次执行先生成任务级登记表：

| 角色 | 路径 | ref | 本次身份 | 是否分析 | 是否修改 |
|---|---|---|---|---|---|
| 承接前台 | path | branch/commit | SOURCE/TARGET/OUT | yes/no | yes/no |

- `SOURCE`：承载已确认改动和证据的项目。
- `TARGET`：用户授权实施迁移的项目。
- `OUT`：不在本次范围，只在必要时说明不分析原因。

目标未明确且不同选择会改变交付范围时，先确认目标集合；不能默认广播到全部六个项目。

登记表还要记录 `delivery_order`、领头分支、派生分支、当前状态、前置门禁、来源会话 ID 和当前会话 ID。未来项目只登记高层范围与待验证差异，不预写可执行实现任务。
