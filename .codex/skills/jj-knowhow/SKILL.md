# jj-knowhow

## 适用场景

用户希望总结 Codex 线程、提交、问题、解决方案或交付模式，供后续任务复用。

## 执行步骤

1. 收集原始对话、提交、文档、测试结果和关键决策。
2. 区分一次性事实、可复用规则、风险陷阱和项目约束。
3. 生成适合沉淀的位置：knowhow、spec、wiki、issue 或 workflow recipe。
4. 保留证据来源和复用条件，避免把猜测写成规则。

## 边界

不要为了总结而创造不存在的流程；知识必须能追溯到真实证据。
