# jj-fix

## 适用场景

用户提供线上现象、错误日志、ARMS/SLS 指纹、时间窗、页面路径或回归描述时使用。

## 执行步骤

1. 锚定用户给出的真实页面、时间窗、错误指纹和影响范围；若问题属于同源项目族，从当前领头项目的分析阶段进入 `$jj-same` 并建立家族交付计划。
2. 查日志、复现路径、相关代码和历史变更，必要时用 blame/log 定位引入点。
3. 输出根因机制、引入位置和修复位置。
4. 做最小范围修复，避免误伤其他租户、站点或旧流程。
5. 验证复现路径、回归点和测试结果。
6. 修复验证通过后，由 `$jj-same` 更新家族交付计划并形成跨会话交接；后续项目仍须重新证明同一根因，不把所有 sibling 自动视为需要修复。

## 边界

不要先抽象推理再找证据；没有日志或复现证据时保持 `PENDING`。
